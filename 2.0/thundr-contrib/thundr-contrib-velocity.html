thundr-contrib-velocity Velocity

A thundr module for rendering views using Apache Velocity.

Velocity is a templating language. Once this module has been included and configured, you can return a VelocityView from any controller method to render content to the request client.

Setting up

Add the module dependency to your build tool and require the module in your ApplicationModule:

pom.xml - Click here to find the latest version

{% highlight xml %} ... com.threewks.thundr thundr-contrib-velocity 2.0.0 compile ... {% endhighlight %}

Your ApplicationModule

{% highlight java %} ... @Override public void requires(DependencyRegistry dependencyRegistry) { super.requires(dependencyRegistry); dependencyRegistry.addDependency(VelocityModule.class); } ... {% endhighlight %}

VelocityView

Once set up, you can return a VelocityView from a controller method to render output to the request client. VelocityView can also be used when sending emails.

{% highlight java %} ... public VelocityView view() { Map model = model(); populateReferenceData(model); return new VelocityView("view.hbs", model); } ... {% endhighlight %}

VelocityView is a template view, so you can set headers, cookies, status code etc on it.

It also respects the Global Model.
