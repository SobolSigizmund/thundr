thundr contributions

Overview

thundr is designed to be modular, which means that additional functionality can be be provided by external parties.

Contrib libraries are libraries contributed to the community by the community.

Contributing

You are free to write and release whatever you want, but we ask that you adhere to the following guidelines to help broadcast to other people the reliability and quality of your modules.

  • To help delineate core thundr libraries from community contributions, non-core libraries should be named thundr-contrib-*. This makes it easier to search for them in maven central, and also makes it clear that versions may trail behind core thundr releases.
  • Contrib modules should be synced to maven central (you can use oss sonatype to do this)
  • Minimally your should have a README available where you host your code (probably github or bitbucket). You can also issue a pull request on the thundr gh-pages branch to integrate into this listing.
  • Version your module using semver (semantic versioning). This is particularly important because modules can have transitive dependencies.
