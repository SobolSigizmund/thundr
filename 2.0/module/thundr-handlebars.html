thundr-handlebars thundr with Handlebars

Overview

Handlebars is a templating language based on the popular Mustache templating language.
You can read about Handlebars in general here.

thundr-handlebars provides the HandlebarsView and HandlebarsViewResolver implementations that allow you to use handlebars templates with thundr.

The handlebars implementation is provided by jknack.

Source on github

Setting Up

Including the thundr-handlebars module will automatically register the HandlebarsViewResovler.

This is accomplished by adding the thundr-handlebars library to your project (or adding the dependency to your maven or similar config). You then need to add a dependency on the HandlebarsModule in your ApplicationModule.

pom.xml - Click here to find the latest version

{% highlight xml %} ... com.threewks.thundr thundr-handlebars 1.1.0 compile ... {% endhighlight %}

Your ApplicationModule

{% highlight java %} ... @Override public void requires(DependencyRegistry dependencyRegistry) { super.requires(dependencyRegistry); dependencyRegistry.addDependency(HandlebarsModule.class); } ... {% endhighlight %}

HandlebarsView

Once you have included the module, you can return a HandlebarsView from any controller method (or provide it for emails) like the following example:

{% highlight java %} ... public HandlebarsView view() { Map model = model(); populateReferenceData(model); return new HandlebarsView("view.hbs", model); } ... {% endhighlight %}

HandlebarsView extends TemplateView, so you can set headers, cookies, status code etc on it.

It also respects the Global Model.

Helpers

The power of Handlebars comes from the ability to register helpers, which execute adhoc code that can be invoked from inside templates.

HandlebarsModule preregisters com.github.jknack.handlebars.helper.StringHelpers for you.

If you wish to register custom helpers, you can do so in your ApplicationModule in configure() or start() by getting the Handlebars instance out of the injection context.

{% highlight java %} ... @Override public void start(UpdatableInjectionContext injectionContext) { super.start(injectionContext); Handlebars handlebars = injectionContext.get(Handlebars.class); handlebars.registerHelper("myHelper", new MyHelper()); } ... {% endhighlight %}
