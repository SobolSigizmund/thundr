thundr glossary

Action

An action defines what happens when an individual route is called by a request. Generally, this invokes a controller method, or serves a static resource.
More

ActionInterceptor

An ActionInterceptor allows custom code to be invoked before, after and after an exception of a controller method.
More

ApplicationModule

The ApplicationModule is the module for your application. It is where you configure your module dependencies and provide your classes and instances into the injection context.
More

Binding

See Data Binding

Controller

A controller is a method on a class which processes a request by executing some logic and returning a view (or otherwise populating the response). Controller interchangeably refers to the class itself, and the method in the class.
More

Data Binding

Data binding usually refers to populating controller method arguments with data from the request. In particular, binding refers to type transformation from request content to specific object types, from collections and lists, to dates, to javabean object graphs.
More

Global Model

The GlobalModel is a map of reference data which is automatically available when rendering all views.
More

Injection Context

The injection context holds all the configuration for dependency injection. You inject instances and classes into it, and it can be used to get instances later, instantiating new ones if necessary. It knows how to invoke constructors and satisfy dependencies.
More

JavaMailMailer

JavaMailMailer is an implementation of the Mailer interface that uses javax.mail
More

Mailer

Mailer defines an interface for sending emails. See JavaMailMailer
More

Module

A module is a self contained component which provides behaviour to an application. The are defined by a Module implementation which injects the module's services into the injection context.
More

Route

A routes is a url pattern used to map a request onto an action.
More

View

A view refers to the object returned from a controller. Views are processed by a view resolver to populate the response.
More

ViewResolver

A ViewResolver writes the response to the HttpServletRequest. The data controlling what to write to the response is held in the View returned from a controller method.
More

ViewResolverRegistry

The ViewResolverRegistry is used to register the ViewResolver which handles a specific type of View
More
